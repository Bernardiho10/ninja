Getting Started

The Ninja JS SDK provides a simple way to integrate National Identification Number (NIN) and Bank Verification Number (BVN) validation into your client-side applications. Our SDK handles the UI, data collection, and secure submission to our verification engines.

Installation

Include the Ninja JS library in your HTML file using our high-performance CDN.

HTML
<script src="https://cdn.ninjahub.com/sdk/v1/ninja.js"></script>

Initialization

Create a new instance of the Ninja class by passing a configuration object. You can choose to render the verification interface as a button, a link, or embed it directly into a container.

Javascript
const ninja = new Ninja({
  apiKey: "YOUR_PUBLIC_API_KEY",
  idType: Ninja.IdType.NIN,
  display: "button",
  targetElement: "#verify-container"
});

ninja.render();

NIN Verification Example

The following example demonstrates how to use the onBeforeSubmit callback to inject data from your own application's UI before the verification process begins.

Example Implementation
// Initialize Ninja for NIN Verification
var ninButton = new Ninja({
    apiKey: "c18aa93a-5df9-41f2-bb75-ninja",
    idType: Ninja.IdType.NIN,
    mode: Ninja.Mode.VERIFY,
    display: "button",
    targetElement: "#nin-button",
    
    onBeforeSubmit: function(input) {
        // Collect data from your own UI before Ninja submits
        input.idNumber = "1234354657";
        input.firstName = "James";
        input.lastName = "Bond";
        input.dateOfBirth = "1988-11-20";
        
        console.log("onBeforeSubmit: injected data into input");
        // return false here to cancel submission if validation fails
    },
    
    onSuccess: function(res) { 
        console.log("NIN Button succeeded: " + JSON.stringify(res)); 
    },
    
    onFailure: function(res) { 
        console.log("NIN Button failed: " + JSON.stringify(res)); 
    },
    
    onError: function(err) { 
        console.error("NIN Button error: " + err.message); 
    },
});

ninButton.render();

API Reference

Configuration Options

The configuration object supports the following properties:

Property Type Description
apiKey String Your public API key retrieved from the Ninja Dashboard.
idType Enum The type of identity to verify. Use Ninja.IdType.NIN or Ninja.IdType.BVN.
mode Enum Verification mode. Defaults to Ninja.Mode.VERIFY.
onBeforeSubmit Function Callback triggered before submission. Useful for data injection.
